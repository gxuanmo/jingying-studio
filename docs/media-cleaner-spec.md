# Media Cleaner Spec v0.1

## 1. 目标

构建一个工具网站，支持用户上传图片或视频，自动或半自动去除以下内容：

- 图片中的文字
- 图片中的水印
- 视频中的静态水印
- 视频中的动态水印
- 视频底部字幕
- 视频中烧进画面的字幕

网站目标不是做“玩具 Demo”，而是做一个可持续迭代的产品原型：

- 第一阶段先追求 `可用 + 可解释 + 可控`
- 第二阶段再追求 `高画质 + 高鲁棒性 + 更强自动化`

## 2. 第一性原理

这个问题的本质不是“擦掉东西”，而是两件事：

1. 精确定位哪些像素不属于原始画面
2. 用画面中其他时空信息重建被遮挡的背景

因此系统一定会拆成两个核心子问题：

- `Masking`: 生成待删除区域的高质量 mask
- `Reconstruction`: 根据上下文进行修复

效果上限通常由这两个环节共同决定，但在真实产品里，`mask 质量` 往往更决定成败。

## 3. 用户场景

### 3.1 图片

- 去截图里的水印
- 去海报、商品图、封面图里的字
- 去带半透明 logo 的图片

### 3.2 视频

- 去角标、台标、logo、水印
- 去片头片尾或中间出现的动态贴纸/水印
- 去底部双语字幕或单语字幕
- 去烧进画面的人名条、注释文字、说明文字

## 4. 产品范围

### 4.1 MVP 范围

- 支持图片上传与下载
- 支持视频上传与下载
- 支持自动检测以下类型：
  - 图片文字
  - 图片水印
  - 视频静态水印
  - 视频底部字幕
- 支持半自动处理以下类型：
  - 视频动态水印
  - 视频烧录字幕
- 支持人工修正：
  - 框选
  - 涂抹 mask
  - 擦除 mask
- 支持结果预览：
  - 原图 / 原视频
  - mask 预览
  - 输出结果

### 4.2 非目标

- 不在第一版承诺“一键处理所有极端视频”
- 不做非法内容处理规避
- 不做 DRM、加密流、直播源抓取
- 不做批量云端大规模渲染平台

## 5. 用户流程

### 5.1 图片处理

1. 用户上传图片
2. 选择任务类型：
   - 去文字
   - 去水印
   - 自动识别全部
3. 系统生成候选 mask
4. 用户可修正 mask
5. 系统执行修复
6. 用户预览并下载

### 5.2 视频处理

1. 用户上传视频
2. 选择任务类型：
   - 静态水印
   - 动态水印
   - 底部字幕
   - 烧录字幕
   - 自动识别全部
3. 系统抽帧并生成候选 mask
4. 用户在时间轴上检查关键帧 mask
5. 系统执行视频修复
6. 用户预览处理结果并下载

## 6. 推荐交互模式

为了避免“算法看起来很聪明，但结果不可控”，推荐交互设计为：

- 默认 `Auto + Manual Fix`
- 自动生成 mask，不直接无提示处理
- 图片显示可编辑 mask 图层
- 视频显示关键帧时间轴与 mask 覆盖层
- 用户可以指定处理区域：
  - 整个底部区域
  - 某个固定角落
  - 手动画一块区域

这个设计会显著提高首版真实可用性。

## 7. 算法路线

## 7.1 整体架构

算法分三层：

1. `Candidate Discovery`
   - 找到疑似文字 / 水印区域
2. `Mask Refinement`
   - 细化到像素级 mask
3. `Inpainting / Reconstruction`
   - 修复背景并保证视频时序稳定

### 7.2 图片算法

#### A. 图片文字去除

推荐方案：

- 文本检测模型：
  - `DBNet / DBNet++`
  - 可选 `CRAFT`
- mask 细化：
  - 二值化 + 连通域 + 膨胀腐蚀
  - 可选文字笔画级细化
- 图像修复：
  - MVP：`OpenCV Telea / Navier-Stokes`
  - 升级版：`LaMa`
  - 高阶版：扩散式 inpainting

#### B. 图片水印去除

按类型拆分：

- 不透明小 logo
- 半透明水印
- 重复平铺水印

推荐策略：

- 先检测规则区域、角标区域、半透明叠加区域
- 对静态角标优先使用：
  - 边缘与透明度异常检测
  - 高频纹理残差检测
- 修复阶段：
  - 小区域用经典 inpainting
  - 大区域优先 LaMa 或扩散模型

## 7.3 视频算法

### A. 视频静态水印

定义：

- 水印位置基本固定
- 形状固定
- 可能半透明

推荐方案：

1. 对视频采样多帧
2. 在同一位置上做时间统计：
   - 时序中值
   - 方差
   - 梯度稳定性
3. 找到长期存在、与背景运动不一致的区域
4. 生成静态 mask
5. 用时序补洞：
   - 邻帧对齐
   - 光流传播
   - 失败时回退到单帧 inpainting

### B. 视频动态水印

定义：

- 位置会移动
- 可能缩放、淡入淡出
- 可能是贴纸、弹幕式浮层或移动 logo

推荐方案：

1. 每帧检测疑似区域
2. 用跟踪器跨帧关联：
   - `Kalman + IoU`
   - `SORT / ByteTrack` 风格轨迹关联
3. 对轨迹上的目标生成时序 mask
4. 使用：
   - 运动估计
   - 前后帧内容传播
   - 时域一致性约束
5. 对难例回退到用户选定 ROI + 半自动模式

### C. 视频底部字幕

特点：

- 高度集中在下方固定带状区域
- 具有明显的水平排布
- 跨帧持续一段时间后切换

推荐方案：

1. 优先限制搜索范围到底部 20% 到 35%
2. 在 ROI 中做文字检测
3. 结合时序停留时间过滤误检
4. 对字幕区域进行统一 mask
5. 用时序背景恢复：
   - 使用前后未出现同字幕的帧做背景候选
   - 无干净帧时再做 inpainting

### D. 视频烧录字幕

特点：

- 可能出现在任意位置
- 常伴随描边、阴影、底色条
- 会遮挡复杂背景

推荐方案：

1. 全帧文字检测
2. 多帧文本跟踪
3. mask 细化到文字笔画与底板
4. 先用时序传播补背景
5. 再做残差修复

这一类是最难的，MVP 不建议承诺全自动高质量。

## 8. 手搓算法建议

为了先做出一版“自己可控”的方案，建议第一版算法不是直接堆大模型，而是采用：

`规则检测 + 轻量跟踪 + 时空修复 + 人工修正`

### 8.1 手搓版核心思路

#### 图片

- 用边缘、连通域、颜色一致性找文字/水印候选区域
- 用形态学操作得到初始 mask
- 小洞用 Telea 修复
- 大洞预留对接 LaMa

#### 视频静态元素

- 对多帧做采样
- 计算每个像素位置的：
  - 平均值
  - 中值
  - 方差
  - 梯度稳定度
- 找到“长期固定出现、但与底层背景运动统计不一致”的区域
- 将该区域作为全局静态 mask

#### 视频动态元素

- 每帧先做局部候选检测
- 将候选框串成轨迹
- 对每条轨迹做 mask union
- 对每帧修复时：
  - 先找前后相邻帧的对应背景
  - 使用光流或块匹配把背景 warp 到当前帧
  - 多候选融合
  - 失败时回退到单帧 inpainting

### 8.2 为什么这种方案适合 MVP

- 可解释，方便 debug
- 不依赖超大模型
- 对底部字幕和静态角标非常实用
- 能为后续模型升级提供稳定的 mask / tracking 框架
- 加上人工修正后，产品可用性会明显提高

## 9. 为什么剪映 / CapCut 效果更好

基于当前行业公开信息和工程经验推断，原因不只是“有算法”，而是完整系统能力更强：

- 检测更准：
  - 不是只做一个框，而是像素级 mask + 时序跟踪
- 视频修复更强：
  - 不是逐帧修，而是利用前后帧时空信息
- 数据更多：
  - 有海量真实视频上的训练与调参数据
- 分场景做优化：
  - 字幕、角标、贴纸、人物遮挡不是同一种策略
- 交互更强：
  - 往往允许用户选区域，减少全自动误伤
- 工程更强：
  - 稳定的编解码、光流、缓存、分辨率分级、失败回退策略

所以它看起来像“一个 AI 功能”，实际上通常是：

`检测 + 分割 + 跟踪 + 时空补全 + 场景路由 + 大量数据调优 + 产品交互`

## 10. 技术架构建议

推荐采用前后端分离：

- 前端：`Next.js`
- 算法服务：`Python + FastAPI`
- 异步任务：`Redis + Worker`
- 存储：
  - 本地开发先用磁盘
  - 线上可换对象存储

### 10.1 前端职责

- 上传文件
- 展示处理模式
- 展示 mask 预览
- 提供手动修 mask 的交互
- 展示任务进度与结果下载

### 10.2 后端职责

- 文件管理
- 抽帧与编码
- 检测 / 跟踪 / 修复任务编排
- 输出结果视频
- 日志与错误诊断

## 13. 质量标准

### 13.1 MVP 验收

- 图片：
  - 小面积文字 / 水印去除后无明显脏块
- 视频静态水印：
  - 固定角标可稳定去除，闪烁可接受但不能严重跳变
- 视频底部字幕：
  - 规则字幕可较稳定去除
- 视频动态水印与烧录字幕：
  - 允许进入半自动模式

### 13.2 失败定义

- 修复区域出现强烈闪烁
- 误删主体内容
- 文字残影明显
- 结果视频无法播放

## 14. 迭代路线

### Phase 1

- 图片文字 / 图片水印
- 视频静态水印
- 视频底部字幕
- 手动 mask 修正

### Phase 2

- 视频动态水印
- 视频烧录字幕
- 关键帧编辑 + 时序传播

### Phase 3

- LaMa / 扩散式修复
- 更强文字检测 / 分割
- 模型路由与质量评分

## 15. 当前建议的产品策略

推荐我们先做：

- `网站 + MVP 算法引擎`
- 优先打透以下三类：
  - 图片文字/水印
  - 视频静态角标
  - 视频底部字幕

原因：

- 这三类最容易先做出靠谱体验
- 可以最快建立上传、预览、mask、处理、下载的产品闭环
- 为后面更难的动态水印和烧录字幕打基础

## 16. 待确认问题

这些问题需要在正式开发前拍板：

1. 第一版是否允许“自动检测 + 手工修正”作为默认工作流
2. 第一版是否优先做本地部署 / 单机版，而不是直接上云
3. 第一版视频是否限制：
   - 分辨率上限
   - 时长上限
   - 文件大小上限
4. 第一版是否把“烧录字幕”和“动态水印”定义为 Beta 功能
5. 第一版是否接受：
   - 图片强自动化
   - 视频强调可控和稳定，不强求全自动
